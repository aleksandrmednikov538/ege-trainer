# ЕГЭ Обществознание — Telegram Mini App

Тренажёр первой части ЕГЭ-2026: **983 задания, 63 темы** по кодификатору.

## Что внутри

```
webapp/
  index.html       ← сам Mini App (одностраничник, 22 КБ)
  questions.json   ← база заданий (1.3 МБ)
bot.py             ← мини-бот, открывает Mini App кнопкой
```

Mini App работает полностью на клиенте. Статистика и список ошибок хранятся в
`Telegram CloudStorage` — это встроенное в Telegram KV-хранилище, синхронится
между всеми устройствами пользователя. **Бэкенда не нужно.**

## Деплой за 10 минут

### 1. Залить статику куда-нибудь с HTTPS

Telegram требует HTTPS для Mini App. Бесплатные варианты:

**GitHub Pages** (проще всего):
```bash
# создай репо, положи туда содержимое папки webapp/
git init && git add . && git commit -m "init"
git remote add origin git@github.com:USER/ege-trainer.git
git push -u origin main
# в Settings → Pages → Source: main / root
# url будет: https://USER.github.io/ege-trainer/
```

Альтернативы: **Cloudflare Pages**, **Vercel**, **Netlify** — всё бесплатно,
любой подойдёт. Главное: чтобы по адресу открывался `index.html`, а рядом
лежал `questions.json`.

### 2. Создать бота

В Telegram: **@BotFather** → `/newbot` → получишь токен вида
`1234567890:AAH...`.

Там же, у `@BotFather`:
- `/setmenubutton` → выбери бота → введи URL → введи текст «Тренажёр».
  После этого у пользователя в чате будет постоянная кнопка-меню для запуска
  Mini App (не только по /start).

### 3. Запустить бот

```bash
pip install aiogram
export BOT_TOKEN="1234567890:AAH..."
export WEBAPP_URL="https://USER.github.io/ege-trainer/"
python bot.py
```

Хостинг бота: подойдёт любой бесплатный — Railway, Fly.io, бесплатный
VPS. На время разработки можно крутить локально.

### 4. Проверить

В Telegram у бота: `/start` → жми кнопку → откроется Mini App.

## Как устроен Mini App

**Три экрана**, переключаются JS, без перезагрузок:

1. **Главная** — стата (решено / верно / точность), три режима:
   *Случайный* / *По теме* / *Работа над ошибками*.
2. **Список тем** — все 63 раздела кодификатора с числом заданий.
3. **Вопрос** — формулировка, варианты, ввод ответа, мгновенная проверка.

**Проверка ответа** учитывает специфику ФИПИ:
- В ответе нет повторяющихся цифр (типы 1, 2, 5–8 и т.п.) → порядок не важен,
  сравниваем как множества.
- В ответе есть повторы (типы 4, 16 — задачи на соответствие) → порядок важен,
  сравниваем как строки.

**Хранилище**: `Telegram.WebApp.CloudStorage` с фоллбэком на `localStorage`
(чтобы можно было дебажить в обычном браузере). Хранится:
- `stats_v1` — счётчики и стрик
- `mistakes_v1` — массив id вопросов, в которых ошибся (для режима «Работа над
  ошибками»; при правильном ответе вопрос автоматически уходит из списка).

**Telegram-интеграции**:
- `MainButton` снизу для «Ответить»/«Дальше» (Telegram сам её красиво
  показывает)
- `BackButton` сверху для возврата на главную
- `HapticFeedback` при верном/неверном ответе
- CSS-переменные темы Telegram — Mini App автоматом подстраивается под
  светлую/тёмную тему пользователя

## Что добавить потом

Идеи на следующие итерации, по убыванию пользы:

1. **ИИ-разбор неправильного ответа.** На экране «не совсем» — кнопка
   «Объяснить». Шлёт вопрос + варианты + правильный ответ в Anthropic API
   (через свой лёгкий бэкенд на FastAPI / Cloudflare Worker), получает разбор
   почему именно эти варианты верны. Это та фишка, которой почти нет у
   конкурентов — стоит ~$0.01 за вопрос.
2. **Режим экзамена**: 25 случайных заданий, без подсказок, в конце оценка.
3. **Прогресс по темам**: на главной — heatmap-таблица 63 тем с цветом по
   точности. Видно, где провал.
4. **Лидерборд среди друзей** через `requestWriteAccess()` + бэкенд.
5. **Закладки** для сложных вопросов.
6. **Парсинг второй части** (типы 17–25) + ИИ-проверка по критериям. Это
   крупная отдельная фича — большинство существующих ботов второй частью
   честно не занимаются.

## Локальная разработка без Telegram

Можно открывать `index.html` через любой статический сервер:
```bash
cd webapp && python3 -m http.server 8000
# открыть http://localhost:8000/
```
В обычном браузере Mini App работает в «дев-режиме»: вместо CloudStorage
используется localStorage, MainButton/BackButton заменены кнопкой снизу.
Можно гонять весь UX без бота.
